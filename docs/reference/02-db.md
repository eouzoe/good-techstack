## Drizzle ORM

**Version**: 0.45.2 stable (v1.0.0-rc.4 pre-release)

Database access layer, running on Workers/D1:

```ts
import { drizzle } from "drizzle-orm/d1";
const db = drizzle(env.DB);
const result = await db.select().from(items).where(eq(items.userId, userId));
```

### Coexistence with better-auth

| Role               | Tool                                                           |
| ------------------ | -------------------------------------------------------------- |
| Auth tables        | better-auth manages these automatically, no schema needed      |
| Application tables | Drizzle ORM, defined manually                                  |
| Migrations         | Drizzle Kit generates them, `wrangler d1 migrations` runs them |

### Why Drizzle over Prisma

| Aspect         | Drizzle                                           | Prisma                       |
| -------------- | ------------------------------------------------- | ---------------------------- |
| Runtime        | Zero overhead (SQL-like)                          | Thick runtime binary         |
| Workers        | `drizzle-orm/d1` native                           | `@prisma/adapter-d1` beta    |
| Bundle         | ~10KB                                             | ~5MB                         |
| Agent-friendly | SQL-like, easy for AI to generate correct queries | Schema DSL requires learning |

### In-memory testing with Bun

```ts
import { Database } from "bun:sqlite";
import { drizzle } from "drizzle-orm/bun-sqlite";
const sqlite = new Database(":memory:");
const db = drizzle(sqlite);
```

No mocks. No Docker.

## D1 (Cloudflare SQLite)

- Serverless SQLite, 5GB free
- Migrations: `wrangler d1 migrations`
- Binding: `env.DB`
- Portability: dump → import (standard SQLite)

## Type-safe chain

```
Zod schema (packages/contract/)
  → Drizzle: z.infer<> maps to DB row type
  → oRPC: .input(UserCreate).output(UserResponse)
  → TanStack Form: zodResolver(UserCreate)
```
